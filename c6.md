# ESP32-C6 Smart Weather & Clock Display

An advanced, high-performance ESPHome firmware configuration for the **ESP32-C6 DevKitC-1** featuring a 128x64 OLED display (`SH1106_128X64`) powered by LVGL version 9 and a **hardware-accelerated algorithmic particle/vector rendering engine** (`include/weather_engine.h`) showcasing the 160MHz RISC-V compute performance of the ESP32-C6.

---

## Showcase Features

- **Debug Mode Animation Showcase**: Enabled by default (`debug_mode: true`). Automatically cycles through all 8 algorithmic weather animations every 5 seconds so you can preview and verify all visual effects without waiting for live weather state updates!
- **Home Assistant Data Ingestion**: Real-time integration with Home Assistant for indoor temperature sensors and weather forecast entity data (`weather.forecast_home`).
- **Dynamic Dual-Page Interface**:
  - **Page 1 (Indoor & Time)**: Top-left indoor temperature statistics and bottom-right digital clock.
  - **Page 2 (Outdoor Forecast)**: Large outdoor temperature readout and forecast condition text.
  - **Auto-Page Rotation**: Smooth page switching loop every 6 seconds.
- **Hardware Interaction & Controls**:
  - 🔄 **Rotary Encoder Weather Selector**: Turn clockwise or counter-clockwise to manually step through weather animation modes (1 to 8). Manual rotation automatically pauses auto-showcase mode so your selected effect stays active.
  - 🔘 **Tactile Push Button Navigation**: Short press (50ms–500ms) instantly toggles between Page 1 and Page 2 and resets the auto-scroll timer. Long press (1s–5s) toggles Debug Showcase Mode on/off.
- **Hardware-Accelerated 60 FPS Engine**: Overclocked I2C bus (`800kHz`) and synchronized hardware display refresh (`update_interval: 16ms`) for ultra-fluid vector animations.
- **Algorithmic Weather FX Showcase (Overlay Canvas)**: Zero UI widget bloat. All weather visuals are procedurally rendered in modular C++ vector math and a particle physics engine onto a single transparent LVGL overlay canvas (`top_layer`):
  - ☀️ **Sunny / Clear Day**: Solid sun disk featuring an expressive cute face with specular-highlighted pupil eyes that smoothly rotate $\pm 30^\circ$ and a cohesively swaying mouth arc, surrounded by 8 dynamic rotating line vectors with pulsating solar flares and atmospheric shimmer motes.
  - 🌙 **Clear Night**: Solid filled crescent moon with geometric lunar crater marks, a whole-screen 24-star multi-phased twinkling starfield with cross-star markers, and procedural shooting star meteors with trailing vector tails.
  - ☁️ **Cloudy / Partly Cloudy**: Organic volumetric cumulus cloud clusters composed of overlapping filled circles with smooth dual-layer parallax drifting and off-screen wrapping.
  - 🌧️ **Torrential Rain / Pouring**: 16 razor-sharp, strictly 1-pixel wide slanted rain streaks rendered via integer-stepped vertical runs to eliminate anti-aliased blurring/worminess, complete with ground impact splash ripples.
  - ❄️ **Snowy Blizzard**: 16 multi-sized snowflakes ($1\times1, 2\times2, 3\times3$) fluttering with dual-axis trigonometric sine turbulence.
  - ⚡ **Thunderstorm**: Multi-branching volcanic lightning bolts with rare, non-blinding 50% scanline dithered atmospheric strobe flashes across dark storm clouds.
  - 💨 **Windy / Fog**: Fast vector stream lines sweeping rightwards with swirling dust particles executing orbital sine-wave drifts.
  - 🌾 **Swaying Grass Horizon**: A dense 28-blade 1-pixel thick grass horizon spanning the bottom edge ($y = 51..63$). Uses spatial traveling wave physics ($\sin(\text{step} \cdot v - gx \cdot k)$) to simulate rhythmic rolling wind-waves like a wheatfield. Rendered with integer-stepped vertical run grouping to guarantee crisp 1px wide strands without anti-aliased thickening on sway, complemented by delicate 3-pixel flowering bulbs atop selected swaying stalks.

## Hardware Specifications

| Hardware Component | Details |
| :--- | :--- |
| **Microcontroller** | Espressif ESP32-C6 DevKitC-1 (RISC-V, 4MB Flash, Wi-Fi 6, BLE 5, Thread/Zigbee) |
| **Framework** | `esp-idf` |
| **Display** | SH1106 / SSD1306 128x64 Monochrome I2C OLED |
| **I2C Pins** | SCL: GPIO 7, SDA: GPIO 6 (`800kHz` frequency) |
| **Rotary Encoder** | EC11 Rotary Encoder (Pin A: GPIO 18, Pin B: GPIO 19) |
| **Push Button** | Single Tactile Switch / Encoder Push Button (SW: GPIO 20) |

### Key Configuration Highlights in `c6.yaml`

#### 1. Hardware & Display Setup
```yaml
i2c:
  - scl: 7
    sda: 6
    id: i2c_1
    frequency: 800khz

display:
  - platform: ssd1306_i2c
    i2c_id: i2c_1
    model: SH1106_128X64
    id: display_ssd1306_i2c_1
    auto_clear_enabled: false
    update_interval: 16ms # Unlocks 60 FPS physical screen flushing
```

#### 2. Rotary Encoder & Push Button Setup
```yaml
sensor:
  - platform: rotary_encoder
    name: "Weather Mode Selector"
    id: weather_mode_encoder
    pin_a:
      number: GPIO18
      mode:
        input: true
        pullup: true
    pin_b:
      number: GPIO19
      mode:
        input: true
        pullup: true
    on_clockwise:
      - lambda: |-
          id(debug_mode) = false;
          id(weather_mode) = (id(weather_mode) % 8) + 1;
    on_anticlockwise:
      - lambda: |-
          id(debug_mode) = false;
          id(weather_mode) = (id(weather_mode) == 1) ? 8 : id(weather_mode) - 1;

binary_sensor:
  - platform: gpio
    name: "Display Navigation Button"
    id: display_button
    pin:
      number: GPIO20
      mode:
        input: true
        pullup: true
    on_click:
      - min_length: 50ms
        max_length: 500ms
        then:
          - if:
              condition:
                lambda: 'return id(current_page) == 1;'
              then:
                - lvgl.page.show: page_two
                - lambda: 'id(current_page) = 2;'
              else:
                - lvgl.page.show: page_one
                - lambda: 'id(current_page) = 1;'
          - script.execute: auto_scroll_pages
      - min_length: 1000ms
        max_length: 5000ms
        then:
          - lambda: |-
              id(debug_mode) = !id(debug_mode);
```

#### 3. LVGL Overlay Canvas & Pages
```yaml
lvgl:
  id: my_lvgl
  displays: display_ssd1306_i2c_1
  update_interval: 16ms
  top_layer:
    widgets:
      - canvas:
          id: weather_canvas
          width: 128
          height: 64
          transparent: true
```

#### 4. Algorithmic Renderer Engine (`interval: 33ms`)
Every 33ms (~30 FPS), a C++ animation lambda advances particle physics, clears transparent canvas buffers, and evaluates dynamic trigonometric formulas for active weather effects. All screen coordinates are strictly bounded to $[0, 127] \times [0, 63]$ for memory-safe LVGL 9 rendering.
